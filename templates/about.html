About the Project

CSM355 Machine Learning Project | Lovely Professional University

Problem Statement

Farmers in India and across the world often struggle to determine which fertilizer is most suitable for their specific soil and crop conditions. Incorrect fertilizer application leads to:

  • Reduced crop yield and poor quality produce
  • Wastage of money on unsuitable fertilizers
  • Soil degradation from excess chemicals
  • Environmental pollution from fertilizer runoff

This project addresses this problem by building a Machine Learning-based web application that recommends the most appropriate fertilizer using Logistic Regression, based on the farmer's soil test results and environmental data.

Objectives

  • Build a Logistic Regression model to classify the best fertilizer
  • Develop a Flask-based web application for easy access
  • Accept soil, crop and environmental inputs from the user
  • Provide a real-time fertilizer recommendation with confidence score
  • Create a user-friendly, responsive frontend using Bootstrap 5
  • Evaluate model performance using accuracy and confusion matrix

Scope of the Project

  • Handles 7 types of fertilizers: Urea, DAP, 14-35-14, 17-17-17, 20-20, 28-28, Super Phosphate
  • Supports 5 soil types: Black, Clayey, Loamy, Red, Sandy
  • Supports 6 crop types: Maize, Paddy, Wheat, Sugarcane, Cotton, Tobacco
  • Processes 8 input features per prediction
  • Deployable locally or on cloud platforms
  • Extendable with more crop/fertilizer types in the future

Why Logistic Regression?

Logistic Regression is a supervised Machine Learning algorithm used for classification problems. Even though the name contains "regression", it is actually used to predict categories (classes), not continuous values.

It works by calculating the probability that an input belongs to each category, then assigning the category with the highest probability as the output. For multi-class problems like fertilizer recommendation, it uses a technique called softmax to handle all classes simultaneously.

Why it is suitable for this project:

  • Simple, fast and easy to interpret
  • Works well for multi-class classification
  • Performs well on structured/tabular datasets
  • Does not require heavy computational resources
  • Ideal for beginner-level academic ML projects

📐 Core Formula (Sigmoid)

P(y=1) = 1 / (1 + e−z)

Where z = w₁x₁ + w₂x₂ + ... + b
